## 상세 풀이

### 견적서 (Estimate / Quotation)

**한 줄 정의**: 시공사가 발주자에게 공사 비용을 제안하는 문서. 공종별 단가·수량·금액 명세를 담은 BOQ 초안.

**상세**:
견적서는 계약 전 단계에서 발주자의 요청에 따라 시공사가 제출한다. 수주를 목적으로 작성되므로 경쟁 견적에서는 단가를 조정하는 경우가 많다. 견적 단계가 끝나고 계약이 성사되면 견적서가 내역서로 확정되며, 이 시점의 합계 금액이 [[03-money#내역서금액-estimate-amount]]이 된다. 견적서와 내역서는 형식이 동일하므로 실무에서는 같은 파일을 견적 단계에서는 "견적서", 계약 후에는 "내역서"라 부르는 경우가 흔하다.

**태그**: #개념:문서 #단계:견적

**동의어 / 별칭**: 견적, 견적금액 문서

**영문**: estimate, quotation

**관련 용어**:
- [[04-documents#내역서-bill-of-quantities]] — 견적 확정 후의 형태
- [[03-money#내역서금액-estimate-amount]] — 견적서 합계 금액

**출처**:
- excel-신사동 (`실행내역서` 시트 — 적산용역 계약 항목)
- excel-지급 (`실행내역서` 시트 — `내역서 제출 금액` 열)

### 내역서 (Bill of Quantities)

**한 줄 정의**: 공종별 재료비·노무비·경비 항목 및 단가·수량·금액을 열거한 공사비 명세서. 계약 체결 후 시공사의 수익 기준 문서.

**상세**:
내역서는 공사를 구성하는 모든 공종을 항목으로 나열하고, 각 항목에 단가·수량·금액을 기재한 BOQ(Bill of Quantities) 문서다. 발주자(클라이언트)에게 제출해 대금을 받는 방향이며, 합계가 [[03-money#내역서금액-estimate-amount]]이 된다.

**신사동.xlsx 구조 — 실행내역서 시트**:
`신사동 626-77 근생 실행내역서` 시트는 하나의 내역서가 아니라 **내역서(수익) + 지급 현황(지출)을 결합한 관리 시트**다. 주요 열:

| 열 | 설명 |
|---|---|
| 내역 | 공종명 (예: 가설공사, 철근콘크리트공사) |
| 실행계약금액 | 해당 공종 하청의 계약금액 (지출 기준) |
| 결재금액 | 이미 지급 완료된 금액 |
| 금회청구액 | 현재 회차에 청구된 금액 |
| Nego. | 네고(할인) 금액 |
| 잔금 | 미지급 잔액 |
| 업체명 | 해당 공종 수행 하청업체 |

신사동 파일에서 `실행내역서`는 상단에 `신사동 626-77 근생 실행내역서`가 있고, 별도 시트로 `신사동 626-77 실행내역서`(공종별 별도 집계)도 존재한다. 실무에서는 두 시트를 묶어 **실행내역서**라 부른다.

**태그**: #개념:문서 #단계:견적 #주의:혼동주의

**동의어 / 별칭**: 실행내역서, 내역, BOQ

**영문**: Bill of Quantities (BOQ)

**주의 / 함정**:
- **내역서 vs 지급내역서**: 이름이 유사하지만 방향이 반대. 내역서는 발주자 → 시공사(수익), 지급내역서는 시공사 → 하청(지출). → [↓ 지급내역서 참조](#지급내역서-payment-statement)
- **내역서 vs 실행내역서**: `실행내역서`는 엑셀 시트 이름으로 자주 등장하며, 순수 내역서(수익 명세)와 지급 현황을 합친 관리 문서를 가리킨다. 두 단어를 혼용하는 실무자가 많음.

**관련 용어**:
- [[03-money#내역서금액-estimate-amount]] — 내역서 합계 금액
- [[04-documents#견적서-estimate--quotation]] — 계약 전 단계의 동일 형식 문서
- [[04-documents#지급내역서-payment-statement]] — 방향이 반대인 혼동 위험 문서

**출처**:
- excel-신사동 (`실행내역서` 시트, `신사동 626-77 실행내역서` 시트)
- excel-지급 (`실행내역서` 시트 — `내역서 제출 금액` 열)
- pdf-기성 (p.6~13 공종별 내역 구조)

### 지급내역서 (Payment Statement)

**한 줄 정의**: 시공사가 하청 업체에 실제 지급한 금액을 차수·일자별로 기록한 문서. 시공사 입장에서 지출을 증명하는 내부 문서.

**상세**:
지급내역서는 **시공사 → 하청** 방향의 지출 기록이다. 어떤 업체에 얼마를, 언제, 어떤 공사 항목으로 지급했는지를 명시한다. 내역서와 이름이 비슷해 혼동이 빈번하지만 돈의 방향이 정반대다.

**방향 비교**:

| 문서 | 돈의 방향 | 시공사 입장 |
|---|---|---|
| 내역서 | 발주자 → 시공사 (청구) | 수익 기준 문서 |
| **지급내역서** | 시공사 → 하청 (지급) | **지출 기준 문서** |

**excel-지급 시트 구조**:
`지급내역서_샘플.xlsx`의 `실행내역서` 시트 주요 열:

| 열 | 설명 |
|---|---|
| 업체명 | 지급 대상 하청업체 |
| 공사내용 | 해당 공종명 |
| 내역서 제출 금액 | 하청이 청구한 금액 |
| 총 공사비 | 계약 기준 총액 |
| 추가공사비 | 변경·추가 공사 금액 |
| 기성누계 | 현재까지 지급 누계 |
| 금회청구 | 이번 회차 지급 요청액 |
| 잔금액 | 미지급 잔액 |
| 지급누계 / 비율 | 누계 지급액 및 비율 |
| 입금 정보 | 수취인 은행·계좌번호·예금주 |

**태그**: #개념:문서 #단계:지급 #주의:혼동주의

**영문**: payment statement, subcontractor payment record

**주의 / 함정**:
- 이름에 "내역서"가 포함되어 있어 발주자 청구 문서(내역서)와 혼동하기 매우 쉬움.
- 신사동 엑셀의 `실행내역서` 시트에는 두 개념이 열 단위로 혼재되어 있음(내역서금액 + 지급 현황 함께 관리).
- MVP 코드에서는 `WeeklyRequest` 타입이 주차별 지급요청 개념을 담고, 지급내역서 자체는 별도 모델링 없음.

**관련 용어**:
- [[04-documents#내역서-bill-of-quantities]] — 방향 반대 (수익 문서)
- [[04-documents#지급요청리스트-weekly-payment-request-list]] — 주차별 요청의 원본 목록
- [[03-money#계약금액-contract-amount]] — 지급내역서의 계약 기준 금액

**출처**:
- excel-지급 (`실행내역서` 시트 전체)
- excel-신사동 (`실행내역서` 시트 결재금액·지급 관련 열)

### 기성청구서 (Progress Billing Document)

**한 줄 정의**: 시공사가 발주자에게 공정 진행분 대금을 청구하기 위해 제출하는 공식 합본 문서. 표지·집계표·원가계산서·공종 내역·공정확인서·현장사진으로 구성.

**상세**:
기성청구서는 단일 양식이 아니라 여러 서류의 **합본(合本)** 이다. 실물 PDF(하이콘 뉴캠퍼스 C동 리모델링공사 4차 기성청구서, 총 20페이지)의 구조:

| 페이지 | 구성 요소 | 역할 |
|---|---|---|
| p.1 | (표지 커버) | 문서 식별 |
| p.2 | 현장명·날짜 | 공식 표지 |
| p.3 | 청구 공문 | 발주자에 보내는 공식 공문 (청구 금액·계좌 명시) |
| p.4 | **기성금액집계표** | 계약금액·전회기성·금회기성·누계기성·대비 한눈에 요약 |
| p.5 | **공사원가계산서** | 재료비·노무비·경비·간접비·이윤 법정 비율 집계 |
| p.6~13 | 공종별 내역 (원가계산서 세부) | 공종 × (재료비·노무비·경비·합계) × (계약·금회·누계) |
| p.14 | **건축공정확인서** | 공종별 공정 진행률(%) 확인, 발주자·시공자 정보 |
| p.15 | 공정 현황 상세 | 건축공정확인서 이어지는 공정 세부 |
| p.16~19 | 현장 사진 | 공종별 시공 사진 (판넬·에어배관·설비·전기) |
| p.20 | (비어 있음) | – |

- 전회기성누계: 2,451,727,774원
- 금회기성청구: 381,438,546원
- 누계기성: 2,833,166,320원
- 청구기간: 2026-03-06 ~ 2026-04-12 / 공정율 93.81%

### 지급요청리스트 (Weekly Payment Request List)

**한 줄 정의**: 현장소장 또는 현장 담당자가 주차(週次)별로 각 하청 업체의 지급 요청 금액을 정리한 현장 내부 목록.

**상세**:
신사동.xlsx에는 주차별 지급요청리스트 시트가 **28개** 있다 (2025-05-26 ~ 2026-04-06). 각 시트는 동일한 구조로, 그 주에 지급 요청한 모든 하청업체의 청구 내용을 담는다. 주요 열:

- 업체명, 공사내용, 내역서 제출 금액, 총 공사비, 추가공사비
- 실행, 기성누계, 해당 주 지급 요청액
- 지급 후 잔금, 입금 정보 (예금주·은행·계좌번호·연락처)

이 시트는 시공사 내부용이며, 지급 결재가 완료되면 결재금액이 `실행내역서`로 이월된다. MVP 코드에서는 `WeeklyRequest` 타입이 이 개념을 모델링한다.

### 공사원가계산서 (Construction Cost Statement)

**한 줄 정의**: 재료비·노무비·경비를 직접·간접으로 분류하고 법정 비율에 따라 산재보험료·고용보험료·일반관리비·이윤을 가산해 공급가액을 산출하는 공식 원가 명세서.

**상세**:
공사원가계산서는 기성청구서 합본 p.5에 포함되며, 계약금액을 원가 항목으로 분해한 법정 양식이다. 주요 비목 구조:

| 대분류 | 소분류 | 비고 (법정 비율) |
|---|---|---|
| 재료비 | 직접재료비, 간접재료비 | |
| 노무비 | 직접노무비, 간접노무비 | 간접노무비 = 직접노무비 × 5.0% |
| 경비 | 기계경비, 산재보험료, 고용보험료, 기타경비, 산업안전보건관리비, 건강보험료, 노인장기요양보험료, 퇴직공제부금비, 연금보험료 | 산재 3.56%, 고용 1.01%, 건강 1.5%, 연금 4.5%, 퇴직공제 2.3% (직접노무비 기준) |
| 일반관리비 | | (재료비+노무비+경비) × 1.0% |
| 이윤 | | (재료비+노무비+경비) × 1.0% |

모든 열은 **계약금액 / 전회까지 누계금액 / 금회기성 청구금액 / 기성 누계금액** 4개 컬럼으로 구성되어 공정 진행에 따른 비목별 집행 현황을 추적한다.

### 거래명세서·세금계산서

**한 줄 정의**: 거래명세서는 납품·서비스 내역 확인용 비법정 서류이며, 세금계산서는 부가세 신고를 위한 법정 서식이다. 실무에서는 거래명세서 → 세금계산서 순으로 발행한다.

**상세**:

**거래명세서**는 납품 또는 서비스 제공 건별로 품목·수량·단가·금액을 기재한 확인서다. 법정 의무 서류는 아니지만 대금 청구·정산 시 세금계산서와 함께 첨부하는 경우가 많다. 건설 현장에서는 자재 납품 또는 소규모 외주 거래 시 주로 사용한다.

**세금계산서**는 공급가액과 세액(VAT 10%)을 명시해 공급자가 공급받는 자에게 발행하는 법정 서류다. 건설 현장의 모든 과세 거래(도급·하도급·자재 구매 등)는 세금계산서를 수수해야 한다.

| 구분 | 거래명세서 | 세금계산서 |
|---|---|---|
| 법적 성격 | 비법정 | 법정 (부가가치세법) |
| 부가세 표시 | 없음 (금액만) | 공급가액 + 세액 분리 표시 |
| 발행 시점 | 납품·서비스 확인 시 | 공급 완료 시 (월 합산 발행 가능) |
| 보관 의무 | 없음 (관행적 보관) | 5년 |

**계산서**는 면세 거래(감리용역·설계용역 등)에서 발행하는 유사 서류로, 부가세가 없다는 점에서 세금계산서와 구분된다.

**태그**: #개념:문서 #단계:지급

**영문**: delivery statement (거래명세서), tax invoice (세금계산서)

**관련 용어**:
- [[03-money#부가세-vat]] — 세금계산서에 표시되는 세목
- 영수증 — 소액 거래 증빙

**출처**:
- excel-신사동 (`실행내역서` — 세금계산서 항목)
- excel-지급 (`실행내역서` — 세금계산서 처리 항목)

### 건축공정확인서

**한 줄 정의**: 공종별 공정 진행률을 발주자·감리자가 확인·서명하는 공문서. 기성 지급의 객관적 근거 서류.

**상세**:
기성청구서 합본의 핵심 첨부 문서 중 하나로, PDF 기성 자료에서는 p.14~15에 위치한다.
공종별 공정율(예: `공정율 93.81%`)을 명시하고 감리자가 직접 확인·서명한 후 발주자가
승인하는 절차로 처리된다. 이 서류 없이는 기성청구를 정식으로 진행할 수 없다.

신사동 현장의 4차 기성청구서에서는 16개 공종 항목별로 공정율을 표기하고 각 항목 옆에
감리·발주자 도장 또는 서명란이 마련되어 있다. 단순 진행 보고가 아니라 **법적 효력을
가진 공정 인증 문서**라는 점이 핵심.

이 문서에서 사용되는 "공정율"은 MVP의 "진행률"(금액축)과 정의가 다르다 — 시간/공정축
완료 비율을 의미한다 ([[05-progress#공정율-construction-progress-rate]] 참조).

**태그**: #개념:문서 #단계:지급

**동의어 / 별칭**: 공정확인서

**관련 용어**:
- [[04-documents#기성청구서-progress-billing-document]] — 합본의 일부로 포함
- [[05-progress#공정율-construction-progress-rate]] — 이 문서의 측정 단위
- 감리자 — 확인·서명 주체

**출처**:
- pdf-기성 (p.14~15)

**주의 / 함정**:
- 발주자·감리자 서명이 없는 공정확인서는 효력 없음 — 신규 시스템에서 서명 상태 추적 필요.
- "공정율"이 진행률과 동일한 의미로 오인되기 쉬움 → 별도 용어로 분리 유지.
